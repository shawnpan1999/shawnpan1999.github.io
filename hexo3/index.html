（三）GitHub图床

发表于 2020-02-01 | 分类于 Hexo博客搭建 |

有的时候看别人的教程，看到一篇巨好的文章，结果图全是裂的。我们可能会气的想摔鼠标。

拒绝裂图，从我做起。于是博主使用 PicGo + GitHub 搭建个人图床。

为什么使用 GitHub 做图床

优点

免费、易用、稳定

配置 GitHub 图床

新建仓库

右上角 + 号 -> New repository

新建一个仓库，选择 Public 。

获取 tokens

右上角头像 -> Settings -> Developer Settings -> Personal access tokens -> Generate new token

Note 项随便填，repo 勾选，完成 Generate token。

完事后把绿框里的 token 抄在小本本上，以后可能见不到她了。

配置 PicGo

安装 PicGo

PicGo 是一款图床应用软件，有了它，我们在上传图片到 GitHub 时就不要一直 pull 啦，岂不美哉？

PicGo下载地址

根据自己系统下载对应的文件，windows 直接下载 .exe 文件即可。

图床配置

左侧 图床设置 -> GitHub图床。

仓库名：写我们刚才创建的仓库地址，只要保留 username/repo 的格式就行了喔。

分支名：默认 master

Token：把我们刚才记下来的一串 Token 输进去

指定存储路径：不填的话就是直接上传到 GitHub 根目录

自定义域名：格式为 https://raw.githubusercontent.com/[用户名]/[仓库名]/[分支] ，可以通过这个域名访问到图片。

点击确定，大功告成，返回上传区即可上传图片了喔。

使用图片

点击相册，选择一张图片按下第一个按钮即可复制图片的链接了，直接作为 Markdown 粘贴进编辑器即可。

定义自己的域名

有些时候，由于薛定谔的墙，某些时间、某些网络下访问 raw.githubusercontent.com 可能需要科学上网。

解决这个方法可以将我们的图床仓库也发布为 GitHub Pages ，从而通过我们自己的域名来访问它。

打开图床仓库，点击设置，设置 GitHub Pages 的 Source 项为主分支。

我们可以看到图床已经发布到了 http://psw.life/imgCloud/ 。此时，就可以使用 http://psw.life/imgCloud/img/xxx.png 来访问图片了。

最后，在 PicGo 里更新我们新的域名，大功告成~
